# PDA 消息兜底提醒 APP — 现场部署检查清单

> 工业 PDA 上能否长期稳定提醒，**一半靠 APP，一半靠部署**。  
> **优先在 APP 权限页用按钮完成配置**，尽量不要自己翻系统菜单。

---

## 一、安装前确认

- [ ] PDA 系统版本为 **Android 7.1**
- [ ] 已确认 **Hi护士 实际包名**
- [ ] 已安装最新 APK（GitHub Actions Artifact）

---

## 二、APP 内一键配置（推荐流程）

打开 APP 后进入 **权限配置** 页，按按钮操作：

### 1. 通知使用权（必须）

1. 点 **「打开通知使用权设置」**
2. 在列表中开启 **PDA消息兜底**
3. 返回 APP，状态应显示 **已开启**

> 此项不开则无法兜底，也不能进主界面。

### 2. 忽略电池优化（建议，可跳过）

1. 点 **「申请忽略电池优化」** → 选允许/不优化
2. 若本机没有该页面或一直「未开启」：勾选 **「本机无电池优化项，跳过」**

### 3. 确认说明（建议）

- 联新 NR510 上系统「应用信息 → 权限」灰色、写「未请求任何权限」是**正常现象**
- 开机自启请到主界面勾选 **「开机自动启动监控」**（类似 Screen Stream）
- 勾选 **「已知悉，可进入主界面」** 即可继续

### 4. 通用入口

- **「打开本应用详情」**：可选；建议在最近任务中锁定本 APP

### 5. 进入主界面

- 通知使用权开启 + 电池就绪/跳过 + 勾选已知悉后，点 **「进入主界面」**
- 若未完成：会 Toast 提示缺哪一项

---

## 三、主界面设置

- [ ] **开机自动启动监控** 建议开启（APP 内开关，无需系统权限页）
- [ ] **监控包名** 填写 Hi护士 实际包名
- [ ] **提醒总开关** 打开
- [ ] **响铃 / 振动 / 锁屏亮屏** 按现场需求开启
- [ ] **通知停留** 建议选 **60 秒** 或 **手动清除**
- [ ] 点击 **测试兜底通知**，确认有声音、振动、兜底通知
- [ ] **保活状态** 显示前台运行；通知监听尽量为「已连接」

---

## 四、仍建议做的现场动作（30 秒）

| 操作 | 说明 |
|------|------|
| 最近任务锁定 | 打开 APP 后，在最近任务中锁定「PDA消息兜底」 |
| Hi护士 同样白名单 | 本 APP 不能替代 Hi护士 自身收消息 |

---

## 五、验收测试

1. APP 内点「测试兜底通知」→ 有声、振、通知
2. 由同事向 Hi护士 发真实消息 → **双通知**（原通知 + 兜底）
3. 锁屏静置后再测
4. 强制停掉 Hi护士且无系统通知 → 本 APP **不承诺**凭空提醒

---

## 六、常见问题

| 现象 | 处理 |
|------|------|
| 进不了主界面 | 必须开通知使用权；电池可勾选跳过；确认说明勾选「已知悉」 |
| 系统权限灰色 | 正常；开机自启用主界面「开机自动启动监控」开关 |
| 滑动花屏 | v1.2 已关硬件加速并去掉 ScrollView；请装最新 APK |
| 用几天后又失效 | 重新锁定最近任务；到权限页重新打开通知使用权 |
| Hi护士 消息进不了系统栏 | 非本 APP 问题，查 Hi护士 与 WLAN |
